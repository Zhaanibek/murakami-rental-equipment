# Симулятор IoT-устройства AeroTrack

Этот скрипт имитирует работу физического трекера арендуемого оборудования, установленного на транспортном средстве или генераторе. Симулятор отправляет телеметрические данные на бэкенд AeroTrack, движется по заданному маршруту в Алматы, считывает команды от сервера (изменение интервала, состояние реле) и отправляет подтверждения их выполнения (ACK).

---

## 🛠️ Требования

Для работы симулятора необходим Python версии 3.8 или выше.

1. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

---

## 🚀 Запуск симулятора

```bash
python simulator.py
```

По умолчанию симулятор будет отправлять данные от имени устройства `velo_almaty_001` на локальный адрес бэкенда Nginx: `http://localhost:8080`.

---

## ⚙️ Настройки через переменные окружения

Вы можете изменить идентификатор устройства или адрес сервера перед запуском скрипта.

### В Windows (PowerShell):
```powershell
# Изменение ID устройства
$env:DEVICE_ID="scooter_almaty_555"

# Изменение адреса сервера (например, если бэкенд запущен локально без Nginx)
$env:SERVER_URL="http://localhost:8000"

python simulator.py
```

### В Linux / macOS:
```bash
DEVICE_ID="scooter_almaty_555" SERVER_URL="http://localhost:8000" python simulator.py
```

## 🔄 Логика работы симулятора

1. **Координаты**: Скрипт имеет предустановленный массив контрольных точек в Алматы. Он плавно интерполирует геопозицию между точками, имитируя движение со скоростью 10–18 км/ч.
2. **Батарея**: 
   * Если реле включено (`ON`) и подключено питание (`isCharging` = `True`), заряд постепенно увеличивается до 100%.
   * Если питание отключено, заряд батареи медленно уменьшается при движении.
   * Если реле выключено (`OFF`), устройство переходит в спящий режим ожидания с минимальным потреблением.
3. **Моточасы**: Каждую итерацию скрипт увеличивает счетчик общего времени работы (моточасы).
4. **Командный цикл**:
   * В ответе от сервера симулятор ищет команды.
   * При получении `SET_INTERVAL` симулятор мгновенно перестраивает свой таймер сна на указанное количество секунд.
   * При получении команды изменения состояния реле, симулятор переключает его локально (меняется скорость, гаснут датчики).
   * Идентификаторы выполненных команд складываются в массив `acked` и отправляются на сервер в следующем пакете телеметрии, после чего сервер удаляет их из очереди отправки.
