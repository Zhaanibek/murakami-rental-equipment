# AeroTrack — Платформа управления арендным оборудованием (MVP)

AeroTrack — это MVP-система управления арендным оборудованием (например, электровелосипедами, скутерами или промышленными IoT-генераторами) в режиме реального времени. Платформа принимает телеметрию с устройств, сохраняет историю геолокации и показателей датчиков в базе данных, предоставляет графическую панель мониторинга с интерактивной картой и поддерживает очередь команд для управления оборудованием (изменение интервала, отключение/включение питания реле).

---

## Быстрый запуск (одна команда)

Убедитесь, что у вас установлен и запущен **Docker Desktop**.

```bash
cd Murakami_TZ
docker compose up --build
```

**Всё!** Одна команда запускает **4 контейнера**:

| Контейнер | Описание | Порт |
| :--- | :--- | :--- |
| `rental_db` | PostgreSQL 15 | внутренний (изолирован) |
| `rental_backend` | FastAPI API-сервер | внутренний (изолирован) |
| `rental_frontend` | Nginx + SPA-дашборд | **http://localhost:8080** |
| `rental_simulator` | IoT-симулятор устройства | — |

Порядок запуска контейнеров автоматически контролируется через healthcheck-зависимости:
`db (healthy) → backend (healthy) → frontend + simulator`

Симулятор перед началом отправки телеметрии ожидает, пока бэкенд полностью стартует, — никаких ошибок при запуске.

➡ Откройте браузер: [http://localhost:8080](http://localhost:8080)  
Устройство `velo_almaty_001` автоматически появится в списке слева через ~10 секунд после старта. Нажмите на него, чтобы увидеть карту, телеметрию и панель управления.

## Симулятор устройства

Симулятор **запускается автоматически** в составе `docker compose` и не требует ручных действий.

Он имитирует движение IoT-устройства по замкнутому круговому маршруту в **Алматы, Казахстан** (вдоль проспекта Абая и улицы Шаляпина). Симулятор считывает команды от сервера: меняет интервал отправки и переключает состояние реле, отправляя подтверждения (ACK) на сервер в следующих пакетах.

**Просмотр логов симулятора:**
```bash
docker logs -f rental_simulator
```

**Запуск дополнительного устройства вручную (опционально):**

Если вы хотите добавить ещё одно устройство, установите зависимости и запустите скрипт отдельно:
```bash
pip install -r simulator/requirements.txt

# Windows PowerShell:
$env:DEVICE_ID="scooter_almaty_777"
$env:SERVER_URL="http://localhost:8080"
python simulator/simulator.py
```

### Локальный запуск без Docker (для разработки)

Если Docker Desktop недоступен, можно запустить локально:

1. **Задайте переменную подключения к БД** (SQLite для простоты):
   ```powershell
   $env:DATABASE_URL="sqlite:///./rental_equipment.db"
   ```
2. **Создайте окружение и установите зависимости**:
   ```bash
   python -m venv .venv
   .venv\Scripts\activate
   pip install -r backend\requirements.txt
   ```
3. **Запустите бэкенд**:
   ```bash
   uvicorn backend.app.main:app --host 127.0.0.1 --port 8000 --reload
   ```
4. **Откройте фронтенд** — файл `frontend/index.html` прямо в браузере.

## 🛠️ Используемые технологии

* **Frontend**: Vanilla HTML5, CSS3 (современная премиум-стилизация с эффектом **Glassmorphism**, адаптивная сетка CSS Grid/Flexbox), Vanilla JS (ES6).
  * **Карты**: [Leaflet.js](https://leafletjs.com/) + бесплатные темно-стилизованные тайлы **CartoDB Dark Matter**.
  * **Иконки**: FontAwesome CDN.
* **Backend**: **FastAPI** (Python 3.12). Высокопроизводительный асинхронный фреймворк с автогенерацией интерактивной документации Swagger UI.
  * **ORM**: SQLAlchemy 2.0 (параметризованные запросы, защита от SQL-инъекций).
  * **Валидация**: Pydantic v2 (строгая валидация типов входящих JSON).
* **Database**: **PostgreSQL 15** (хранение текущих состояний устройств, логов телеметрии и очереди команд).
* **Docker & Orchestration**: Docker, Docker Compose (Nginx reverse-proxy, FastAPI backend, PostgreSQL db).

## Переменные окружения (`.env`)

Все параметры загружаются из файла `.env` в корне проекта:

| Переменная | Значение по умолчанию | Описание |
| :--- | :--- | :--- |
| `POSTGRES_USER` | `rental_user` | Имя пользователя базы данных PostgreSQL |
| `POSTGRES_PASSWORD` | `rental_secure_pass_2026` | Пароль к базе данных |
| `POSTGRES_DB` | `rental_equipment_db` | Имя создаваемой базы данных |
| `POSTGRES_HOST` | `db` | Имя хоста БД (внутри Docker сети указывает на контейнер `db`) |
| `POSTGRES_PORT` | `5432` | Внутренний порт PostgreSQL |
| `DATABASE_URL` | `postgresql://...` | Строка подключения ORM к PostgreSQL |
| `BACKEND_HOST` | `0.0.0.0` | IP-адрес для прослушивания бэкендом |
| `BACKEND_PORT` | `8000` | Внутренний порт бэкенда |

## API Спецификация

Интерактивная интерактивная документация API (Swagger UI) доступна при запуске проекта по адресу: **`http://localhost:8080/api/docs`** (через прокси) или напрямую на бэкенде: `http://localhost:8000/docs`.

### 1. Отправка телеметрии устройства (API телеметрии)
* **Метод**: `POST`
* **Эндпоинт**: `/api/telemetry/{device_id}`
* **Описание**: Физические устройства отправляют пакеты данных на этот эндпоинт. Бэкенд обновляет текущее состояние устройства, сохраняет координаты в лог перемещений, отмечает выполненные команды и возвращает список новых команд.
* **Пример входящего JSON**:
  ```json
  {
    "recordedAt": "2026-05-16T11:30:00.000Z",
    "lat": 43.238949,
    "lng": 76.889709,
    "battery": 87,
    "powerSource": "battery",
    "isCharging": true,
    "speed": 12.5,
    "accuracy": 8,
    "payload": {
      "operatingMode": "active",
      "hydrogenPpm": 1.2,
      "totalHours": 1834.5
    },
    "acked": [
      "cmd_abc123"
    ],
    "relayState": "ON",
    "mcc": 401,
    "mnc": 1,
    "lac": 13500,
    "cellId": 123456789,
    "rssi": -85
  }
  ```
* **Пример ответа бэкенда**:
  ```json
  {
    "interval": 30,
    "commands": [
      {
        "id": "cmd_8f4a21cd",
        "type": "SET_INTERVAL",
        "payload": {
          "interval": 15
        }
      }
    ],
    "desiredPowerState": "ON",
    "powerCommand": "TURN_ON"
  }
  ```

### 2. Получение списка устройств (Панель мониторинга)
* **Метод**: `GET`
* **Эндпоинт**: `/api/devices`
* **Ответ**: Возвращает список всех зарегистрированных устройств с их актуальными показателями телеметрии.

### 3. Отправка команды устройству
* **Метод**: `POST`
* **Эндпоинт**: `/api/devices/{device_id}/commands`
* **Пример тела запроса**:
  ```json
  {
    "type": "SET_INTERVAL",
    "payload": {
      "interval": 15
    }
  }
  ```
* **Ответ**: Добавляет команду в очередь со статусом `PENDING`. Команда будет доставлена на устройство при следующем сеансе связи.

## Безопасность проекта

В рамках реализации MVP заложены лучшие практики безопасности:
1. **Защита от SQL-инъекций**: SQLAlchemy ORM использует строго типизированные параметризованные запросы. Передача сырого SQL исключена.
2. **Валидация входных данных**: Строгий разбор JSON на основе Pydantic-моделей. Невалидные типы, инъекции в полях или попытки передать неверный интервал отсекаются на уровне парсинга (HTTP 422).
3. **Изоляция сети**: База данных Postgres не публикует свои порты наружу. Бэкенд взаимодействует с ней по внутренней сети Docker. Бэкенд также спрятан за Nginx. Публично доступен только порт Nginx (`8080`), что предотвращает прямое сканирование API.
4. **Безопасность CORS**: Использование Nginx в качестве единой точки входа (reverse proxy) избавляет от необходимости открывать бэкенд через `Access-Control-Allow-Origin: *`.

## Что реализовано

* **Автоматическая инициализация**: База данных автоматически разворачивает схемы таблиц при первом старте контейнера бэкенда.
* **Логика командного цикла (IoT Command Loop)**: Реализован полный цикл "запрос от UI -> очередь PENDING -> отправка SENT -> применение на устройстве -> отправка ACK устройством -> отметка ACKNOWLEDGED в БД".
* **Красивый дашборд**:
  * Темный футуристичный стеклянный интерфейс (Glassmorphism).
  * Интерактивная карта Leaflet, отображающая маркер устройства, статусную пульсацию и историю траектории (точечные вейпоинты с затуханием прозрачности и пунктирная линия пути).
  * Индикатор уровня заряда батареи с цветовой индикацией критического разряда.
  * Анимированная шкала сотовой связи (RSSI).
  * Форматированный просмотрщик RAW JSON последней телеметрии устройства с возможностью копирования в один клик.
* **Автоматические тесты**: Покрытие интеграционными тестами (`test_api.py`) логики работы бэкенда с эмуляцией базы данных через SQLite.

## Направления для улучшения (Future Road)

1. **Авторизация и JWT**: Внедрение ролевой модели управления (администраторы, менеджеры аренды, клиенты) и авторизация по токенам JWT.
2. **WebSocket / SSE**: Переход от периодического опроса (polling) фронтендом на постоянные соединения WebSocket или Server-Sent Events для мгновенного обновления карты и датчиков без лишних HTTP-запросов.
3. **Использование TimescaleDB**: Для хранения миллионов телеметрических записей оптимизировать Postgres с помощью расширения TimescaleDB (для эффективного сжатия временных рядов).
4. **Аналитика поездок**: Расчет пройденной дистанции, средней скорости, времени в пути и детекция геозон (Geofencing) — например, предупреждение при выезде устройства за пределы города Алматы.
